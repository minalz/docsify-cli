```txt


Redis ZSet 天然适合：
┌─────────────────────────────────────────────────────────────┐
│  ZADD time:wheel 1721647200000 "order:12345"               │
│       ↑ key        ↑ score(执行时间戳)    ↑ member(任务ID)  │
│                                                             │
│  ZRANGEBYSCORE time:wheel 0 1721647200000 LIMIT 0 100     │
│  查询 score <= 当前时间的任务，只取到期的                      │
│                                                             │
│  ZREM time:wheel "order:12345"                             │
│  原子移除，确保只有一个实例能抢到                            │
└─────────────────────────────────────────────────────────────┘



┌─────────────────────────────────────────────────────────────────┐
│                         用户视角                                │
│  下单成功 → 页面显示"29:59"倒计时 → 倒计时到0 → 订单自动关闭    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                        实际技术实现                              │
│                                                                 │
│  Layer 1: 前端倒计时（体验层）                                   │
│  ─────────────────────────────                                  │
│  前端从服务器获取"剩余时间"，本地JS倒计时                        │
│  倒计时到0时：前端自动变灰"已关闭"，但**不真正调用取消API**       │
│  目的：提升用户体验，减少无效请求                                │
│                                                                 │
│  Layer 2: Redis ZSet 时间轮（精准触发层）                        │
│  ─────────────────────────────────────                          │
│  下单时：ZADD order:timewheel expireTimestamp orderId            │
│  100ms扫描：到期立即触发取消                                     │
│  误差：< 100ms                                                  │
│                                                                 │
│  Layer 3: 兜底扫描（容错层）                                     │
│  ─────────────────────────                                      │
│  每30秒全表扫描：SELECT * FROM order WHERE status=UNPAID        │
│                AND create_time < NOW() - INTERVAL 30 MINUTE     │
│  目的：防止 Layer 2 漏掉（Redis故障、任务丢失等）                │
│                                                                 │
│  Layer 4: 对账补偿（最终一致性）                                  │
│  ──────────────────────────────                                 │
│  每日T+1对账：系统订单状态 vs 支付流水                           │
│  发现"已支付但订单已关闭" → 人工介入或自动退款补偿               │
└─────────────────────────────────────────────────────────────────┘

```
