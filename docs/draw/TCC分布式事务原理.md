## 📝 说明

> 💡 TCC 是一种两阶段补偿型事务模式：Try 阶段预留资源，Confirm 阶段确认提交，Cancel 阶段取消回滚，适用于对一致性要求较高的分布式业务场景

| 阶段                          | 说明                                                        |
| --------------------------- | --------------------------------------------------------- |
| **Phase 1: Try**            | 事务协调器调用所有参与者的 Try 方法，预占资源（创建待支付收费单、冻结药品库存、预占耗材、创建待完成就诊记录） |
| **决策节点**                    | 所有 Try 是否成功？                                              |
| **Phase 2a: Confirm**（全部成功） | 真正执行业务：更新已支付、扣减库存、确认耗材、生成就诊记录 → 事务完成 ✅                    |
| **Phase 2b: Cancel**（任一失败）  | 回滚释放资源：删除收费单、释放冻结库存、释放耗材、作废处方 → 事务回滚 ❌                    |
