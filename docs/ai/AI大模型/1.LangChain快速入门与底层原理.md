## 📖 简介

**LangChain** 是一个开源的 Python AI 应用开发框架，它提供了构建基于大模型的 AI 应用所需的模块和工具。通过 LangChain，开发者可以轻松地与大型语言模型 (LLM) 集成，完成文本生成、问答、翻译、对话等任务。

> 💡 **核心价值**：LangChain 降低了 AI 应用开发的门槛，让任何人都可以基于 LLM 构建属于自己的创意应用。

### ✨ LangChain 特性

| 特性 | 说明 |
|:---|:---|
| **🤖 LLM 和提示（Prompt）** | 对所有 LLM 大模型进行 API 抽象，统一访问接口，提供 Prompt 提示模板管理机制 |
| **🔗 链 (Chain)** | 对常见场景封装现成模块（如问答系统、SQL 生成等），像工作流一样逐步执行 |
| **⚡ LCEL** | LangChain Expression Language，新版本核心特性，用于灵活自定义工作流编排 |
| **📚 数据增强生成 (RAG)** | 导入新信息增强 LLM 生成质量，弥补模型无法回答新问题的短板 |
| **🎯 Agents** | 基于大模型的应用设计模式，自动调用外部系统完成任务 |
| **💭 模型记忆（Memory）** | 让大模型记住之前的对话内容 |

## 🏗️ LangChain 框架组成

![image-1753858752963](http://img.minalz.cn/typora/image-1753858752963.svg)

LangChain 框架由以下几个部分组成：

### 1. 📚 LangChain 库
Python 和 JavaScript 库，包含接口和集成多种组件的运行时基础，以及现成的链和代理的实现。

### 2. 📋 LangChain 模板
官方提供的一些 AI 任务模板，快速启动项目。

### 3. 🚀 LangServe
基于 FastAPI，可以将 LangChain 定义的链 (Chain) 发布成为 REST API。

### 4. 🔍 LangSmith
开发平台（云服务），支持 LangChain debug、任务监控。

## 📦 LangChain 库 (Libraries)

LangChain 库本身由几个不同的包组成：

| 包名 | 说明 |
|:---|:---|
| `langchain-core` | 基础抽象和 LangChain 表达语言 |
| `langchain-community` | 第三方集成，主要包括集成的第三方组件 |
| `langchain` | 主要包括链 (chain)、代理 (agent) 和检索策略 |

## 🔄 LangChain 任务处理流程

![image-1753858753082](http://img.minalz.cn/typora/image-1753858753082.jpeg)

如上图，LangChain 提供一套提示词模板 (prompt template) 管理工具，负责处理提示词，然后传递给大模型处理，最后处理大模型返回的结果。

### 模型封装类型

LangChain 对大模型的封装主要包括两种类型：

| 类型 | 说明 |
|:---|:---|
| **LLM** | 问答模型，接收文本输入，返回文本结果 |
| **Chat Model** | 对话模型，接收一组对话消息，返回对话消息（类似聊天） |

## 🎯 核心概念

### 1. LLMs
LangChain 封装的基础模型，模型接收一个文本输入，然后返回一个文本结果。

### 2. Chat Models
聊天模型（或对话模型），与 LLMs 不同，这些模型专为对话场景而设计。模型可以接收一组对话消息，然后返回对话消息。

### 3. 消息（Message）
指的是聊天模型的消息内容，消息类型包括：
- `HumanMessage` - 用户消息
- `AIMessage` - AI 回复消息
- `SystemMessage` - 系统消息
- `FunctionMessage` - 函数消息
- `ToolMessage` - 工具消息

### 4. 提示 (Prompts)
LangChain 封装了一组专门用于提示词管理的工具类，方便格式化提示词内容。

### 5. 输出解析器 (Output Parsers)
接受大模型返回的文本内容后，可以使用专门的输出解析器对文本内容进行格式化，例如：
- 解析 JSON
- 将 LLM 输出转换为 Python 对象

### 6. Retrievers
为方便将私有数据导入到大模型 (LLM)，提高模型回答问题的质量，LangChain 封装了检索框架 (Retrievers)，方便我们：
- 📄 加载文档数据
- ✂️ 切割文档数据
- 💾 存储和检索文档数据

### 7. 向量存储 (Vector stores)
为支持私有数据的语义相似搜索，LangChain 支持多种向量数据库。

### 8. Agents
智能体 (Agents)，通常指的是以大模型 (LLM) 作为决策引擎，根据用户输入的任务，自动调用外部系统、硬件设备共同完成用户的任务。

## 🚀 快速入门

### 1️⃣ 安装 LangChain

使用 Pip 安装：

```bash
pip install langchain
```

### 2️⃣ 初始化模型

在使用 LangChain 之前，需要导入 LangChain x OpenAI 集成包，并设置 API 密钥。

**获取 API 密钥：**
1. 创建 OpenAI 账户
2. 访问 [API Keys](https://platform.openai.com/account/api-keys) 获取密钥

**设置环境变量：**
```bash
export OPENAI_API_KEY="YOUR_API_KEY"
```

**初始化模型：**
```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI()
```

### 3️⃣ 使用 LLM

使用 LLM 回答问题非常简单，可以直接调用 `invoke` 方法。还可以通过提示模板 (prompt template) 生成提示词。

**构建简单的 LLM 链：**

```python
from langchain_core.prompts import ChatPromptTemplate

# 创建提示模板
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是世界级的技术专家"),
    ("user", "{input}")
])

# 基于 LCEL 表达式构建 LLM 链
# 类似 Linux pipeline 语法，从左到右按顺序执行
chain = prompt | llm

# 调用 LLM 链
chain.invoke({"input": "帮我写一篇关于AI的技术文章，100个字"})
```

### 4️⃣ 输出转换

LLM 的输出通常是一条消息，为了更方便处理结果，可以将消息转换为字符串。

```python
from langchain_core.output_parsers import StrOutputParser

# 创建字符串输出解析器
output_parser = StrOutputParser()

# 将输出解析器添加到 LLM 链中
chain = prompt | llm | output_parser

# 调用 LLM 链
chain.invoke({"input": "帮我写一篇langchain的技术文章，100个字"})
```
